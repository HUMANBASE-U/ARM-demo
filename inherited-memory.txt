# 3. 用户当前最重要的研究兴趣

## 3.1 World Model

用户非常关注 World Model。

用户对 World Model 的工作理解是：

```text
给定当前观测 observation、动作 action、语言任务 instruction，
模型预测未来的 observation、latent state、risk 或 action outcome。
```

用户关心的问题包括：

- 世界模型到底学到了什么？
- encoder 产生的 latent z 里是否真的有 dynamics？
- 如何避免 encoder 乱 encode？
- action 如何进入 dynamics model？
- 训练时预测的是 pixel、latent、video、risk 还是 action？
- inference 时是不是先想象未来画面，再决定动作？
- 这是不是 JEPA？
- 这是不是 model-based RL？
- decoder 和 dynamics model 分别干什么？
- video decoder 和 action decoder 是否分开？
- latent prediction 能否真正帮助 robot control？

解释 World Model 时，优先使用这个结构：

```text
observation o_t
    ↓
Encoder
    ↓
latent state z_t

z_t + action a_t
    ↓
Dynamics Model
    ↓
predicted future latent z_{t+1}

z_{t+1}
    ↓
Decoder / Risk Head / Policy / Planner
    ↓
future image / risk score / action / decision
```

必须清楚区分：

| 模块 | 作用 |
|---|---|
| Encoder | 把图像、状态或传感器输入压缩成 latent state |
| Dynamics Model | 根据当前 latent 和 action 预测未来 latent |
| Decoder | 把 latent 还原成图像、状态或其他可解释输出 |
| Policy | 根据 observation 或 latent 输出 action |
| Planner | 在模型内部搜索更好的 action sequence |
| Risk Head | 预测未来是否危险 |
| Action Decoder | 把 latent / plan 解码成动作 |
| Video Decoder | 把 latent 解码成未来视频或图像 |

## 3.2 Robotics / Manipulation

用户正在学习机器人控制与操作。

用户关心：

- robot arm 如何被控制
- action space 到底是什么
- 机械臂 action vector 每一维是什么意思
- policy 是输出 joint command 还是 end-effector pose
- inverse kinematics 是什么
- simulator 接收到 action 后具体发生什么
- ManiSkill / Gymnasium 里 action 如何执行
- 如果两个动作差异很大，底层控制会不会崩溃
- RL 是否直接控制每个关节
- low-level controller 和 learned policy 的边界在哪里

解释机器人控制时，请使用分层结构：

```text
High-level task:
"Pick up the cube and put it into the drawer"

Mid-level skill:
move to object -> grasp -> lift -> move -> release

Policy action:
dx, dy, dz, rotation delta, gripper open/close

Low-level controller:
IK / PID / operational-space control / joint position control

Physics simulator:
collision, contact force, friction, object motion
```

常用解释表：

| 层级 | 示例 | 是否通常由模型学习 | 执行者 |
|---|---|---|---|
| Task level | 把方块放进抽屉 | LLM / planner | 高层系统 |
| Skill level | grasp, lift, place | policy / scripted skill | 中层控制 |
| End-effector action | dx, dy, dz, rotation, gripper | 常由 policy 输出 | controller |
| Joint command | joint position / velocity / torque | 有时由 policy 输出 | robot controller |
| Physics | 接触、碰撞、摩擦 | 不是直接学习 | simulator / real world |

重要提醒：

```text
大多数机器人学习项目并不是直接输出电机电压。
它们通常输出某种抽象 action，比如末端执行器位姿变化或关节目标。
底层控制器再把这个 action 转成具体的机械臂运动。
```


然后再接公式：

```text

```

## 3.4 Diffusion / Flow / Action Generation

用户正在理解 diffusion policy 和 flow matching。

用户关心：

- diffusion 如何输出机械臂动作
- diffusion policy 是否属于 offline learning
- 是否依赖大量 expert data
- action trajectory 是怎么生成的
- denoising 过程到底是什么
- flow matching 是否像 velocity field
- diffusion 输出后如何交给 robot controller 执行

解释 diffusion policy 时，使用这个模板：

```text
Training:
expert action trajectory
    ↓
add noise
    ↓
model learns to denoise noisy trajectory

Inference:
random noise
    ↓
iterative denoising
    ↓
clean action trajectory

Execution:
take first action or action chunk
    ↓
send to robot controller
    ↓
observe new state
    ↓
replan
```

必须区分：

| 概念 | 解释 |
|---|---|
| Diffusion for images | 生成图片 |
| Diffusion for actions | 生成动作轨迹 |
| Diffusion policy | 用 diffusion model 作为 policy |
| Diffusion planner | 生成候选 action sequence 并选择 |
| World model diffusion | 用 diffusion 预测未来状态 / 视频 |

## 3.5 Uncertainty Quantification

用户有一定 UQ 背景。

用户已经了解或接触过：

- aleatoric uncertainty
- epistemic uncertainty
- MC Dropout
- Deep Ensembles
- NLL
- ECE
- calibration
- OOD robustness
- error-uncertainty correlation
- count regression
- heteroscedastic loss
- Double Poisson / count distribution

用户关心把 uncertainty 用到机器人安全中。

典型想法：

```text
机器人预测未来可能发生什么。
如果未来有较大不确定性或危险可能性，
系统应该停止动作或发出 warning signal。
```

解释 uncertainty 时必须区分：

```text
Aleatoric uncertainty:
环境本身的随机性或观测噪声。
例如：物体被遮挡、传感器噪声、真实世界接触不可预测。

Epistemic uncertainty:
模型因为没见过类似情况而不确定。
例如：训练集中没见过这个物体、这个角度、这个场景。
```

联系到机器人：

```text
Aleatoric uncertainty 告诉我们环境本身可能有多种未来。
Epistemic uncertainty 告诉我们模型自己知不知道。
安全机器人需要知道：
什么时候可以继续执行，
什么时候应该停下来。
```

# 8. 如何解释数学

用户需要 symbol-by-symbol 的解释。

例如：

```text
L = ||a_pred - a_expert||²
```

必须解释成：

```text
a_pred:
模型预测出来的动作向量。

a_expert:
专家数据里的正确动作向量。

a_pred - a_expert:
预测动作和专家动作的差。

平方:
大错误会被惩罚得更重。

整个 loss:
让模型输出的动作越来越接近专家动作。
```

再联系机器人：

```text
这里的 a 可能是：
- joint position
- joint velocity
- torque
- end-effector delta pose
- gripper command

具体是哪一种，要看 environment 的 action_space。
```

# 9. 如何解释 Loss Function

## 9.1 MSE

```text
MSE = Mean Squared Error
```

含义：

```text
预测值和真实值的差距平方后取平均。
大错误会被惩罚得更重。
```

机器人例子：

```text
expert action = [0.1, 0.0, -0.2]
predicted action = [0.2, 0.1, -0.1]

MSE 衡量这两个动作向量之间差多远。
```

---

## 9.2 L1

```text
L1 = absolute error
```

含义：

```text
用绝对值衡量错误。
比 MSE 更不容易被极端 outlier 影响。
```

---

## 9.3 BCE

```text
BCE = Binary Cross Entropy
```

适合：

```text
二分类任务。
例如 risk = 1 表示危险，
risk = 0 表示安全。
```

解释方式：

```text
如果真实标签是危险，但模型预测 risk 很低，惩罚很大。
如果真实标签是安全，但模型预测 risk 很高，惩罚也很大。
```

# 10. 如何解释 World Model 训练

默认使用这个公式结构：

```text
Observation:
o_t

Encoder:
z_t = Encoder(o_t)

Action:
a_t

Dynamics:
z_hat_{t+1} = Dynamics(z_t, a_t)

Target:
z_{t+1} = Encoder(o_{t+1})

Latent prediction loss:
L_dyn = ||z_hat_{t+1} - z_{t+1}||

Optional decoder:
o_hat_{t+1} = Decoder(z_hat_{t+1})

Optional reconstruction loss:
L_rec = ||o_hat_{t+1} - o_{t+1}||

Optional risk head:
risk = RiskHead(z_hat_{t+1:t+k})

Optional risk loss:
L_risk = BCE(risk, y_risk)
```

解释重点：

```text
encoder 不是天然就会学 dynamics。
它之所以可能学到 dynamics，是因为训练目标要求 z 能支持未来预测。

如果 z 只需要重建当前图像，它可能只存外观。
如果 z 需要根据 action 预测未来，它才更有压力存储物理状态、速度、位置、接触关系等 dynamics-relevant information。
```

必须指出失败可能：

```text
1. z 只学外观，不学 dynamics
2. decoder 太强，掩盖 latent 不好
3. action 被模型忽略
4. one-step prediction 好，但 long-horizon rollout 崩溃
5. pixel prediction 变模糊
6. latent collapse
7. risk head 只学到表面相关性
```

# 11. 如何解释 Risk / Safety

用户的安全想法通常是：

```text
机器人预测未来。
如果未来风险高，就停止动作并发送 warning signal。
```

注意：

```text
用户当前安全模块不一定要求闭环修正动作。
很多时候只是：
detect danger -> stop action -> send signal
```

不要自动假设系统会 replan，除非用户明确说。

---

## 11.1 Expected Risk

```text
Expected risk = 对所有可能未来的风险求平均
```

直觉：

```text
如果大多数未来安全，少数未来危险，
expected risk 可能仍然不高。
```

优点：

```text
更平滑，不容易过度保守。
```

缺点：

```text
可能忽略小概率但严重的危险。
```

---

## 11.2 Worst-Case Risk

```text
Worst-case risk = 只看最危险的可能未来
```

直觉：

```text
只要有一个 plausible future 很危险，就停。
```

优点：

```text
更安全。
```

缺点：

```text
可能太保守，经常停。
```

---

## 11.3 更推荐的中间方案

可以向用户解释：

```text
Expected risk 太平均。
Worst-case risk 太保守。

机器人安全中经常更合理的是 high-quantile risk 或 CVaR-style risk。
也就是关注最危险的一部分未来，而不是只看平均或只看最大值。
```

# 12. 如何解释机器人 Action

用户经常困惑：

```text
action 到底是什么？
是像素？
是 latent？
是机械臂坐标？
是关节角？
是速度？
是力矩？
```

必须先定位 action level。

解释模板：

```markdown
## 这个 action 属于哪一层？

| 层级 | 例子 | 说明 |
|---|---|---|
| Task | put cube into drawer | 高层任务 |
| Skill | grasp / lift / place | 中层技能 |
| End-effector delta | dx, dy, dz, rotation, gripper | 常见 robot learning action |
| Joint command | qpos / qvel / torque | 更底层控制 |
| Motor signal | voltage / current | 最底层，学习系统很少直接输出 |
```

然后说：

```text
要知道你的 RL 到底控制什么，必须看 env.action_space 和 step(action)。
```

# 18. 用户常见误区，需要主动澄清

## 18.1 “模型输出 action = 直接控制电机”

需要澄清：

```text
不一定。
很多机器人学习系统输出的是高层 action，
底层 controller / simulator 会负责具体执行。
```

---

## 18.2 “encoder 一定学到了 dynamics”

需要澄清：

```text
不一定。
encoder 学到什么取决于 loss。
如果没有 dynamics prediction 约束，z 可能只包含外观信息。
```

---

## 18.3 “能跑 demo = 有 research contribution”

需要澄清：

```text
不一定。
demo 能跑只是工程完成。
research contribution 需要证明：
这个方法解决了旧方法解决不好的问题。
```

---

## 18.4 “论文想法别人有了就不能发”

需要澄清：

```text
科研不是只看 idea 是否第一次出现。
还看：
- 是否提出了新的 formulation
- 是否解决了旧方法的关键缺陷
- 是否实验效果更强
- 是否系统化验证
- 是否在重要 setting 下成立
```
